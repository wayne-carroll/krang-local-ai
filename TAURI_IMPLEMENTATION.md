# KRANG Tauri Implementation Spec

> Companion to `TAURI_PLAN.md`. The plan says *what* and *why*; this spec says
> *exactly how* - file contents, command signatures, config values, and the
> build order. All 7 decisions in the plan are resolved; this doc assumes them.
>
> Status: ready to implement. Nothing here is built yet.

## 0. Ground rules
- The React frontend chat/streaming logic does not change. The only frontend
  additions are: a `platform.js` shim, a transport swap behind it, and a small
  first-launch status overlay.
- All desktop logic is isolated in `src-tauri/`. Web mode never imports it.
- Free path only: no code signing, no paid accounts, no bundled binaries.

## 1. Target toolchain / versions
- Tauri 2.x (`@tauri-apps/cli@^2`, `@tauri-apps/api@^2`, crate `tauri = "2"`).
- Rust stable via `rustup` (install in Phase 0).
- Xcode Command Line Tools (`xcode-select --install`) - required to compile Rust
  on macOS and to build the DMG.
- Node 25 / npm 11 (already present).
- Plugins: `@tauri-apps/plugin-http`, `@tauri-apps/plugin-shell`,
  `@tauri-apps/plugin-process` (+ matching crates).

## 2. CORS / transport decision (the key technical call)
The Tauri WebView origin is `tauri://localhost` (macOS), which Ollama's default
CORS allowlist (localhost/127.0.0.1 only) rejects. Two-part solution:

1. **Primary:** in desktop mode, route Ollama HTTP through
   `@tauri-apps/plugin-http`'s `fetch` (requests go through the Rust HTTP stack,
   so browser CORS does not apply). Its API mirrors the web Fetch API including
   streaming response bodies, so `ollama.js` keeps using `body.getReader()`.
2. **Belt-and-suspenders:** for any Ollama instance *we* launch, also set
   `OLLAMA_ORIGINS=*` in its environment.

`ollama.js` gets its `fetch` from `platform.js` (`getFetch()`), which returns the
plugin fetch in Tauri and `window.fetch` in the browser. Single choke point.

> Verify at implementation time that `@tauri-apps/plugin-http` streams NDJSON
> incrementally (it should in Tauri 2). If streaming regresses, fall back to
> browser `fetch` + ensuring `OLLAMA_ORIGINS` includes `tauri://localhost`.

## 4. Frontend specifics

### src/lib/platform.js
```js
// Single source of truth for "are we running inside Tauri?" and for the HTTP
// transport. In the browser everything falls back to web globals, so the same
// build runs in both modes.

export function isTauri() {
  return typeof window !== 'undefined' && '__TAURI_INTERNALS__' in window
}

// In Tauri, use the http plugin's fetch (bypasses WebView CORS). In a browser,
// use the global fetch. Same Fetch API surface either way.
let _fetch = null
export async function getFetch() {
  if (_fetch) return _fetch
  if (isTauri()) {
    const { fetch: tauriFetch } = await import('@tauri-apps/plugin-http')
    _fetch = tauriFetch
  } else {
    _fetch = window.fetch.bind(window)
  }
  return _fetch
}

// Thin command wrapper: invoke a Rust command in Tauri, no-op (null) in browser.
export async function invokeCmd(cmd, args) {
  if (!isTauri()) return null
  const { invoke } = await import('@tauri-apps/api/core')
  return invoke(cmd, args)
}

// Subscribe to a Rust event (e.g. ollama setup progress). Returns unlisten fn.
export async function onEvent(name, handler) {
  if (!isTauri()) return () => {}
  const { listen } = await import('@tauri-apps/api/event')
  return listen(name, (e) => handler(e.payload))
}
```

### ollama.js change (pattern, applied to every call site)
Replace the bare `fetch(...)` calls with a resolved transport:
```js
import { getFetch } from './platform.js'
// inside each function, before the request:
const f = await getFetch()
const res = await f(`${OLLAMA_BASE}/api/tags`)   // etc.
```
No other logic changes; streaming, error unwrapping, and stats stay identical.

### src/components/StartupGate.jsx (Tauri-only)
- On mount (only when `isTauri()`), call `invokeCmd('ensure_ollama_ready')`.
- Subscribe via `onEvent('ollama-setup', ...)` to show phase + percent
  ("Checking Ollama", "Downloading Ollama 42%", "Starting Ollama", "Ready").
- Render a full-screen overlay reusing existing empty-state styling until the
  command resolves; on error show the message + a Retry button.
- In the browser this component renders nothing and never calls the backend
  (web mode keeps today's behavior: it just tries to reach Ollama and shows the
  existing `OllamaOfflineError` state).

### App.jsx change
Wrap the existing app so `StartupGate` gates first render in desktop mode only.
No change to chat, history, settings, or model logic.

## 6. Rust backend (src-tauri/src/ollama.rs)

Constants:
```rust
const OLLAMA_BASE: &str = "http://localhost:11434";
// The standalone `ollama-darwin` asset was removed from recent releases (it now
// 404s); the macOS bundle ships as a .tgz containing the binary + runtime libs.
const OLLAMA_DARWIN_URL: &str =
    "https://github.com/ollama/ollama/releases/latest/download/ollama-darwin.tgz";
```

Commands (all `#[tauri::command] async`):

- `is_ollama_running() -> bool`
  GET `{OLLAMA_BASE}/api/tags` with a short timeout; true on 2xx.

- `is_ollama_installed(app) -> bool`
  Check, in order: `which ollama` on PATH; `/Applications/Ollama.app/Contents/
  Resources/ollama`; the app data `bin/ollama`. Return the first that exists.

- `start_ollama(app) -> Result<(), String>`
  Resolve the ollama binary path (same order as above), spawn `ollama serve`
  with env `OLLAMA_ORIGINS=*` via tauri-plugin-shell, store the child handle in
  Tauri managed state (so we can kill only ours on exit), then poll
  `is_ollama_running` up to ~20s. Error string on timeout.

- `install_ollama(app) -> Result<(), String>`
  Stream-download the `.tgz` (reqwest, bytes_stream) to
  `app_data_dir/ollama-darwin.tgz`, emitting `ollama-setup` progress events
  `{ phase: "download", percent }`. Then extract the whole bundle into
  `app_data_dir/bin/` via the system `tar -xzf` (the binary needs its sibling
  libs), remove the archive, `chmod 0o755` the binary, run
  `xattr -dr com.apple.quarantine <bin_dir>` (ignore failure), and return.

- `ensure_ollama_ready(app) -> Result<(), String>` (orchestrator)
  ```
  emit phase "check"
  if is_ollama_running(): emit "ready"; return Ok
  if !is_ollama_installed(): emit "download"; install_ollama()
  emit "start"; start_ollama()
  emit "ready"; Ok
  ```
  Every failure returns a human string surfaced by StartupGate.

Managed state for the child:
```rust
struct OllamaProcess(Mutex<Option<CommandChild>>);
```

### main.rs / lib.rs
- `main.rs`: `fn main() { krang_lib::run() }`.
- `lib.rs`: build the app, register plugins (`http`, `shell`, `process`),
  `.manage(OllamaProcess(Default::default()))`, `.invoke_handler(generate_handler![
  is_ollama_running, is_ollama_installed, start_ollama, install_ollama,
  ensure_ollama_ready])`, and a `RunEvent::ExitRequested` handler that kills the
  child **only if we spawned it** (leave a pre-existing system Ollama running).

## 8. scripts/build-mac.sh
```bash
#!/usr/bin/env bash
set -euo pipefail
cd "$(dirname "$0")/.."
command -v cargo >/dev/null || { echo "Rust not installed. Run: rustup"; exit 1; }
xcode-select -p >/dev/null 2>&1 || { echo "Run: xcode-select --install"; exit 1; }
npm install
npm run tauri:build -- --target aarch64-apple-darwin
echo "DMG:"; find src-tauri/target -name "*.dmg" -print

## 12. Verification matrix
- `npm run dev` in a browser: unchanged behavior, no Tauri calls fire.
- `npm run tauri:dev` with Ollama already running: app skips straight to chat;
  streaming works through plugin-http.
- `tauri:dev` with Ollama installed but stopped: app starts it, then chat works.
- `tauri:dev` with Ollama moved aside: app downloads + installs + starts it
  (progress events visible), then chat works.
- App quit kills only an Ollama we launched; a pre-existing one keeps running.
- `scripts/build-mac.sh` produces a launchable DMG; fresh install works with
  Ollama absent (first-launch flow runs).
- `./start.sh` unchanged on web; `./start.sh --dev` runs the Vite dev server.

## 13. Risks / gotchas (watch these during build)
- **plugin-http streaming:** confirm incremental NDJSON delivery (section 2
  fallback if not).
- **Ollama asset URL:** RESOLVED. `ollama-darwin` 404s now; using
  `ollama-darwin.tgz` (~143 MB) + tar extraction. Tested end to end. Re-verify
  the asset name if Ollama changes their release layout again.
- **Quarantine:** downloaded binary must have `com.apple.quarantine` stripped or
  it will not execute; also it must be executable (`chmod 755`).
- **First-run Gatekeeper:** the app itself is unsigned -> right-click Open once.
- **Fonts/CSP:** Google Fonts preconnect must be allowed by CSP or fonts break.
- **Rust build time:** first `tauri build` is slow (cold crate compile); expected.
